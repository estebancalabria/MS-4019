# Transformación de procesos empresariales cotidianos con agentes

# Uso compartido, colaboración e interacción con agentes en Microsoft 365

## Introducción

Este módulo explora cómo crear, compartir, administrar e interactuar con agentes dentro del ecosistema de Microsoft 365, especialmente en:

* Microsoft 365 Copilot Chat
* SharePoint
* Microsoft Teams

Los agentes impulsados por inteligencia artificial permiten automatizar tareas, acceder rápidamente a información organizacional y mejorar la colaboración entre equipos sin requerir conocimientos técnicos avanzados.

Los agentes pueden:

* Responder preguntas.
* Resumir documentos.
* Automatizar flujos de trabajo.
* Guiar usuarios.
* Centralizar conocimiento.
* Integrarse con Teams y SharePoint.
* Facilitar procesos empresariales diarios.

---

# ¿Qué son los agentes?

Los agentes son asistentes inteligentes configurados para ayudar en:

* Sitios de SharePoint
* Bibliotecas de documentos
* Chats de Copilot
* Equipos y proyectos específicos

Pueden trabajar utilizando como origen de conocimiento:

* Archivos
* Carpetas
* Sitios
* Páginas
* Bibliotecas documentales
* Contextos de chat

Existen:

* Agentes preconfigurados de SharePoint
* Agentes precompilados de Microsoft
* Agentes personalizados creados por usuarios

# Licencias y permisos

Para utilizar agentes:

* Los usuarios necesitan licencias válidas de Microsoft 365.
* El acceso depende de permisos sobre los recursos utilizados.

Los agentes respetan:

* Permisos de SharePoint
* Seguridad organizacional
* Etiquetas de confidencialidad
* Configuración de acceso

* Información a la que el usuario tiene acceso.

# Compartir agentes

## Beneficios del uso compartido

Compartir agentes permite:

* Extender capacidades de IA a otros usuarios.
* Mejorar productividad organizacional.
* Automatizar procesos colaborativos.
* Centralizar soporte y conocimiento.

---

# Opciones de uso compartido en Copilot Chat

| Opción                               | Descripción                    |
| ------------------------------------ | ------------------------------ |
| Cualquier usuario de la organización | Todos pueden acceder           |
| Usuarios específicos                 | Solo ciertos usuarios o grupos |
| Solo tú                              | Solo el creador puede usarlo   |

## Recomendación

Microsoft recomienda:

* Compartir mediante grupos de seguridad.

---

# Restricciones importantes

Los usuarios compartidos:

* Pueden usar el agente.
* NO pueden editarlo.

---

# Compartir agentes en Copilot Chat

Los agentes pueden compartirse:

* Durante la creación.
* Después de publicados.

Se pueden distribuir mediante:

* Outlook
* WhatsApp
* Facebook
* LinkedIn

---

# Compartir automáticamente archivos y carpetas

Cuando un agente usa archivos de SharePoint:

* Los archivos pueden compartirse automáticamente.

## Dependencias

Esto depende de:

* Permisos del creador
* Configuración de SharePoint
* Etiquetas de confidencialidad

## Importante

Si un archivo no puede compartirse:

* El agente sigue funcionando.
* Pero no podrá usar ese contenido.

---

# Diferencias entre Copilot Chat y SharePoint

## Copilot Chat

* El creador puede compartir directamente.
* No requiere aprobación administrativa.

## SharePoint

* Requiere aprobación del administrador o propietario del sitio.
* Ofrece mayor gobernanza y control.

---

# Aprobación de agentes en SharePoint

## Flujo general

### 1. Creación

El usuario crea el agente.

### 2. Envío

Se envía para aprobación.

### 3. Revisión

El administrador verifica:

* Seguridad
* Permisos
* Relevancia
* Precisión

### 4. Aprobación o rechazo

El agente puede:

* Aprobarse.
* Rechazarse con comentarios.

### 5. Publicación

Los agentes aprobados aparecen en:

* Copilots aprobados
* Activos del sitio

---

# Agentes aprobados automáticamente

Si el agente es creado por:

* El propietario del sitio
* Un administrador

Entonces:

* Se aprueba automáticamente.

---

# Agente predeterminado

SharePoint permite:

* Definir un agente predeterminado para el sitio.

# Agregar agentes a Teams

## Proceso

### 1. Iniciar sesión

En SharePoint o Copilot Chat.

### 2. Abrir Copilot

Desde el sitio o chat.

### 3. Seleccionar agente

Buscar agente personalizado.

## Importante

No puede compartirse:

* El agente listo para usar de SharePoint.

### 4. Agregar a Teams

Seleccionar:

* “Agregar a Teams”

### 5. Compartir en chat

Enviar vínculo en Teams.

### 6. Confirmar

Seleccionar:

* “Agregar a este chat”

Después:

* El agente puede utilizarse mediante @mention.

---

# Limitación actual de Teams

Actualmente:

* Solo puede existir un agente por chat.

Agregar otro:

* Reemplaza el anterior.

---

# Quitar agentes de Teams

## Pasos

1. Abrir chat en Teams.
2. Abrir menú de opciones.
3. Seleccionar:

   * “Administrar aplicaciones”
4. Elegir:

   * “Quitar”

---

# Interacción con agentes en SharePoint

## Cómo usar un agente

### 1. Iniciar sesión

En SharePoint.

### 2. Abrir Copilot

Seleccionar el ícono en la esquina superior derecha.

### 3. Elegir agente

Puede:

* Seleccionar agentes aprobados.
* Elegir agentes recientes.

### 4. Hacer preguntas

Ejemplos:

* Resumir documentos.
* Obtener contactos.
* Consultar actualizaciones.

También pueden hacerse:

* Preguntas de seguimiento.

---

# Interacción con agentes en Copilot Chat

# Conclusión

Los agentes de Microsoft 365 representan una evolución importante en la automatización y colaboración empresarial. Gracias a su integración con SharePoint, Copilot Chat y Microsoft Teams, las organizaciones pueden centralizar conocimiento, automatizar tareas y mejorar la experiencia de trabajo diaria.

Copilot Chat ofrece flexibilidad para crear y compartir agentes rápidamente, mientras que SharePoint aporta gobernanza, seguridad y control administrativo. Teams complementa este ecosistema permitiendo integrar agentes directamente en conversaciones y flujos colaborativos.

La correcta administración de permisos, orígenes de conocimiento y políticas de gobernanza es clave para aprovechar todo el potencial de la inteligencia artificial dentro de Microsoft 365.
